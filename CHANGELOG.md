## 🔄 版本发布流程

### 1. 版本准备

```bash
# 更新版本号
npm version [major|minor|patch]

# 更新变更日志
# 编辑CHANGELOG.md文件，添加新版本条目

# 提交版本变更
git add .
git commit -m "chore(release): v0.1.0"
```

### 2. 创建发布

```bash
# 创建Git标签
git tag -a v0.1.0 -m "Release v0.1.0"

# 推送标签到远程
git push origin v0.1.0
```

### 3. GitHub发布

在GitHub上创建新版本发布，包含：

- 版本号
- 变更日志摘要
- 构建的二进制文件
- 安装说明

## 🤝 贡献指南

### 提交消息规范

所有提交消息必须遵循以下格式：

```
<type>[optional scope]: <description>

[optional body]

[optional footer]
```

**Type类型：**
- `feat`: 新功能
- `fix`: 错误修复
- `docs`: 文档更新
- `style`: 代码格式化
- `refactor`: 代码重构
- `test`: 添加测试
- `chore`: 构建过程或辅助工具的变动

**示例：**

```
feat(agent-card): add status indicator

- 添加了智能体状态指示器
- 支持自定义状态颜色

Closes #123
```

### 变更日志更新

当提交新功能或修复错误时，需要在CHANGELOG.md中添加相应的条目：

1. **新功能**: 在对应版本的"新增功能"部分添加
2. **错误修复**: 在对应版本的"错误修复"部分添加
3. **破坏性变更**: 在对应版本的"破坏性变更"部分添加，并标记为**BREAKING CHANGE**
